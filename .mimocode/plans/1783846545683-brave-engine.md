# big_deal 代码整理计划

## 问题诊断

经过全面审查，发现以下核心问题：

### 1. `main.js` 大量重复代码
13 个按钮处理器完全相同的模式：`threads.start → keepScreenOn → require module → run → cancelKeepingAwake`。应提取为一个通用启动函数。

### 2. 函数在多个文件中重复定义
| 函数 | 出现位置 |
|------|----------|
| `switchTag` | `utils.js`, `jd_utils.js`, `jd_bline_box.js` |
| `getEntryBar` | `jd_utils.js`, `jd.js` |
| `homePageGetEnter` | `utils.js`, `jd_utils.js`, `jd.js` |
| `search` | `jd.js`, `jd_night.js` |
| `find_entry` + `find_goto_button` | `jdjr_earn_pea.js`, `jdjr_save_center.js` |

### 3. `utils.js` 混杂通用和京东专用函数
`switchTag`、`getJDHomePageEntry` 是京东专用逻辑，不应在通用工具中。

### 4. 互动游戏模块重复
`jdjr_interactive_games.js` 和 `jdjr_interactive_game_obj.js` 包含相同的 `igtasks` 数组、`enterInteractiveGames`、`jdjriFindPlayButton`、`findTextInChildren` 函数。`_obj` 版本只是 OOP 包装。

### 5. 根目录混杂测试文件
`test.js`、`test_lightning.js`、`zfb_video.js` 是调试/独立脚本，混在主项目中。

### 6. 死代码和注释代码
- `autojs-utils.js`：仅含一个 greeting 函数，疑似未使用
- `jd_super_star.js`：大段注释掉的旧代码（98-166行）
- `sign.js`：注释掉的 smzdm 函数（20-49行）
- `jd_day_day_benefit.js`：第55行有 bug（`obj` 未定义）

---

## 整理方案

### Step 1: 消除 `main.js` 重复代码

提取通用启动函数，将 13 个重复的按钮处理器简化为一行调用：

```js
function startTask(taskFn) {
  threads.start(() => {
    device.keepScreenOn();
    console.log("已开启屏幕常亮");
    taskFn();
    device.cancelKeepingAwake();
    console.log("已关闭屏幕常亮");
  });
}
```

每个处理器变为：
```js
ui.sign.click(() => startTask(require("./modules/sign").run));
ui.jd.click(() => startTask(require("./modules/jd").run));
// ...
```

### Step 2: 统一京东导航函数到 `jd_utils.js`

将 `switchTag`、`getEntryBar`、`homePageGetEnter`、`search` 统一到 `jd_utils.js`，从 `utils.js` 和各模块中移除重复定义。

**`jd_utils.js`** 保留/接收：
- `switchTag`（来自 `utils.js`）
- `getEntryBar`（来自 `jd_utils.js`，已有）
- `homePageGetEnter`（来自 `jd_utils.js`，已有）
- `search`（从 `jd.js` 和 `jd_night.js` 统一）
- `backHomePage`（已有）

**从 `utils.js` 移除**：`switchTag`、`getJDHomePageEntry`

### Step 3: 统一 `find_entry` / `find_goto_button` 到 `jdjr_utils.js`（新建）

`jdjr_earn_pea.js` 和 `jdjr_save_center.js` 中重复的 `find_entry` + `find_goto_button` 提取到新文件 `modules/jdjr_utils.js`。

### Step 4: 合并互动游戏模块

将 `jdjr_interactive_game_obj.js` 合并到 `jdjr_interactive_games.js`。OOP 版本逻辑相同但有 bug（第50行引用了 `e.name` 应为 `this.name`），保留函数式版本，删除 `_obj` 文件。

共享的数据和函数统一放一份：
- `igtasks` 数组（保留一份）
- `enterInteractiveGames`
- `jdjriFindPlayButton` + `findTextInChildren`

### Step 5: 清理根目录

将测试/独立脚本移入 `scripts/` 目录：
- `test.js` → `scripts/test.js`
- `test_lightning.js` → `scripts/test_lightning.js`
- `zfb_video.js` → `scripts/zfb_video.js`

### Step 6: 删除无用文件

- 删除 `modules/autojs-utils.js`（未被任何文件引用）

### Step 7: 清理死代码

- `jd_super_star.js`：删除注释掉的旧 `run` 函数（98-166行）
- `sign.js`：删除注释掉的 `smzdm` 函数（20-49行）
- `jd_day_day_benefit.js`：修复第55行 bug（`obj` → `order`）
